# v4 Oracle Layout Audit: Resettable Integrator

This v4 pilot task separates the public feedback testbench, private score
testbench, and private checker binding.

## Agent-Visible Surface

- `instruction.md`: canonical natural-language task contract.
- `public_contract.json`: machine-readable public contract and oracle policy.
- `test_feedback/public_tb.scs`: public feedback testbench deck for agentic debugging.
- `test_feedback/run_feedback.py`: queryable feedback command for agentic modes.

The feedback command requires an explicit candidate directory:

```bash
VABENCH_FEEDBACK_SOURCE_DIR=<candidate-dir> python3 test_feedback/run_feedback.py
```

## Evaluator-Only Surface

- `solution/`: gold Verilog-A sources for evaluator certification.
- `evaluator/score_tb.scs`: private score testbench deck, not agent-queryable.
- `evaluator/checker_profile.json`: private checker binding and trace contract.
- `evaluator/run_score.py`: evaluator-side score runner.
- `negative_variants/`: checker-certification negative examples where present.

## Migration Provenance

- Public feedback TB migrated from `tasks/013-resettable-integrator/test_visible/tests/tb_visible_smoke.scs`.
- Private score TB migrated from `tasks/013-resettable-integrator/test_hidden/tests/tb_resettable_integrator_ref.scs`.
- Checker binding was split out of the old mixed oracle profiles.
- `starter/` was removed from the model-visible surface; candidates must be generated by the model.

## Remaining Pilot Boundary

This migration changes packaging and oracle access boundaries only. It does not
strengthen the underlying behavior checker or claim a new Spectre final-judge run.
