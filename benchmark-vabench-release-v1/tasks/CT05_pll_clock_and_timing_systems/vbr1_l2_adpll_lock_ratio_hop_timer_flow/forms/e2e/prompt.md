## Public L2 Behavior Contract

This row is an ADPLL lock and ratio-hop flow. It must expose the timing loop,
divider relation, control monitor, and lock behavior:

1. Timer-driven oscillator:
   - Drive `vout` as a voltage-domain DCO clock generated by event/timer
     behavior.
   - Keep the oscillator frequency bounded by the public parameter range.

2. Programmable divider:
   - Interpret `ratio_ctrl` as a programmable divider ratio in the public
     range.
   - Drive `fb_clk` as the divided feedback clock derived from `vout`, not as
     an independent scripted source.
   - Before the ratio hop, the public waveform should show the expected pre-hop
     ratio between `vout` and `ref_clk`.
   - After the ratio hop and settling, the waveform should show the new
     post-hop ratio.
   - The evaluator also checks that `vout` divided by `fb_clk` matches the
     commanded divider ratio, and that `fb_clk` tracks `ref_clk` after lock.

3. Lock and control monitor:
   - Drive `vctrl_mon` as a bounded normalized control-code monitor.
   - Update the control code from measured feedback/reference timing or period
     relation. Do not implement lock by directly mapping `ratio_ctrl` to a
     precomputed target DCO frequency while ignoring `fb_clk`.
   - Drive `lock` high after enough consecutive feedback/reference period
     matches, and reacquire lock after the ratio step.

Use a public testbench that drives a stable reference clock and steps
`ratio_ctrl` from about 4 to about 6 during the transient.

## Task: adpll_ratio_hop_smoke

Write a pure voltage-domain Verilog-A behavioral ADPLL that uses a timer-driven DCO and a programmable feedback divider.

Requirements:

1. Ports must be `electrical` only.
2. The module must take a reference clock input and an analog `ratio_ctrl` input whose rounded value sets the divider ratio in the range 2 to 16.
3. The DCO frequency must be adjusted by an internal digital control code so the divided feedback clock tracks the reference.
   The control update and lock decision must consume feedback timing from `fb_clk`;
   do not bypass the loop by using only `ratio_ctrl` to select a fixed DCO frequency.
4. Expose:
   - `vout` as the DCO clock output
   - `fb_clk` as the divided feedback clock
   - `vctrl_mon` as a normalized monitor of the control code
   - `lock` as a streak-based lock indicator
5. The reference testbench will step `ratio_ctrl` from 4 to 6 during transient.

Expected behavior:
- When ratio_ctrl ≈ 4V: vout frequency / ref_clk frequency ≈ 4.0 (ratio = divider setting)
- When ratio_ctrl ≈ 6V: vout frequency / ref_clk frequency ≈ 6.0 after ratio hop
- In both windows, vout frequency / fb_clk frequency should match the divider ratio
  and fb_clk frequency should match ref_clk frequency after lock.
- lock signal should be ≥ 80% high before and after the ratio change
- vctrl_mon should stay within [0, 1.2V] range
Ports:
- `VDD`: inout electrical
- `VSS`: inout electrical
- `ref_clk`: input electrical
- `ratio_ctrl`: input electrical
- `fb_clk`: output electrical
- `vout`: output electrical
- `vctrl_mon`: output electrical
- `lock`: output electrical

Write pure voltage-domain behavioral Verilog-A with no current contributions.
